# Design & Development Sync Process (Figma Collaboration)

## Process Overview

- **Drafting:** Low-fidelity wireframes to define information architecture and flow.
- **High-fidelity:** Complete main interfaces and key motion frames; organize components and variants.
- **Delivery (Handoff):** Export slices/sequences, attach annotations and motion specs, and organize the asset list.
- **Integration (Tuning):** Developers implement according to specs; verify color values/font sizes/motion timing and Haptics.

## Figma Collaboration Key Points

- Use `Components + Variants` to manage states and sizes; unify naming and Tokens.
- Add `Spec Overlay` to artboards: Color values, font sizes, glow intensity, margins, constraints.
- **Asset List:** Static PNGs, Motion PNG sequences, and necessary short video references.
- **Version Control:** Create a `Release Page` and `Changelog` for every major change.

## Sync Cadence

- **Daily Standup:** Update design progress and blockers.
- **Weekly Iteration:** Freeze a version of motion and static drafts; developers pull for implementation.
- **Regression & Acceptance:** Check item by item against PRD acceptance criteria.
